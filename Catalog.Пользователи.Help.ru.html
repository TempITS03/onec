Справочник «Пользователи»

Данный справочник содержит информацию о пользователях текущей информационной базы. Здесь содержатся основные настройки пользователя. Настройки разбиты по закладкам и в зависимости от роли пользователя необходимо их заполнить.

"Оповещения"

Содержит реквизиты:

  • Флажок "Оповещать при создании нового задания" - при взведении данного флага система переодически будет отслежевать список "Актуальные задачи для оповещения" и если для текущего пользователя, список будет содержать данные, то будет отображено всплывающее окно информируя о поступилении новых задач.
  • "Таймаут (секунд)" - количество секунд для переодического опрашивания списка "Актуальных задач для оповещения"

Важно. Имеет смысл использовать данный вид оповещения только если в текущей информационной базе, задачи будут ставить разные пользователи или же по Web-интерфейсу контрагенты. При обмене данными задачами режим оповещения не работает.

"Почтовый клиент"

Данная закладка содержит необходимые реквизиты для отправки и получения почты.

Важно. Имеет смысл использовать если в системе ведется обмен данными задачами через почту. При этом обмен данными смотрит на константу "Использовать обмен данными через почту" и если константа взведена, то будут опрашиватся текущие почтовые настройки. Если почтовые настройки не заполнены или система по данным настройкам не сможет обменятся данными то пользователю будет выдано соответствующее сообщение.

"Параметры заполнения документов"

Данные реквизиты по-умолчанию будут подстановлены в документ "регламентное задание".

Содержит реквизиты:

  • "Организация"
  • "Основной куратор"
  • "Основной исполнитель"

"Web-интерфейс клиента"

Данные реквизиты по-умолчанию будут подстановлены в документ "регламентное задание", а также используются для распознания пользователя является-ли данный пользователь клиентом. Пользователь является клиентом лишь в случае, когда у него установлена роль "Клиент".

Содержит реквизиты:

  • "Контрагент"
  • "Контактное лицо контрагента"

Важно. Если пользователь является клиентом, то поле "Контрагент" необходимо обязательно заполнить, в противном случае клиент не сможет создать задачу.
